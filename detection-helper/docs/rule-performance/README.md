# Rule Performance Tracker

> **Purpose.** Post-deployment tracking for detection rules. Records alert volume,
> true/false positive rates, tuning history, and retirement decisions. Agents read
> this before suggesting similar rules — if a pattern was retired for high FP, they
> suggest a different approach.

## Directory structure

One file per deployed rule: `<rule-id>.md`

## Status definitions

| Status | Meaning | Agent action |
|---|---|---|
| **active** | Rule working as expected | Can reference as successful example |
| **tuned** | Rule adjusted for FP reduction | Note tuning when suggesting similar |
| **retired** | Rule no longer deployed | Avoid suggesting same pattern without modification |

## How the agent uses this

When Path C (detection authoring) considers a pattern:
1. Check `docs/rule-performance/` for rules using the same `pattern_source` or `primitive`
2. If any are `retired`, read the retirement reason
3. If `tuned`, note the tuning in FP guidance for the new rule
4. If `active` with low FP, use as reference for quality scoring
